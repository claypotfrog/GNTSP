<h3>Notes on Filters</h3>
<ul><li>Filter names are sorted alphabetically in the drop-down menus except for the "Default" filter which is always first.  If you want your filters to appear in any sort of order, name them alphabetically.  Or put a number as their first character because numbers sort before letters.  But remember that numbers are still sorted alphabetically.  So "1" is followed by "10" is followed by "2" alphabetically.</li>
<li>When you filter on Person/Number, this does not affect infinitives since infinitives don't have person or number.  So you might want to include another filter as well to bring out that the infinitives are also verbs.  Like set ALL verbs to show in <b>bold</b> (set all three voices to display in bold - that will filter ALL verbs), but then change the text color or highlight color for the Person/Number.</li>
<li>Since participles don't have person (but do have number), if you filter on Person/Number, then Participles will be filtered according to the 3rd person filters.  So singular participles will be filtered like 3rd singular verbs, and plural participles will be filtered like 3rd plural verbs.  You can see that in the Person/Number screen.  Notice that the two 3rd person lines also say "Ptcp" (for "participle").</li>
<li type="skip"><img src="binfile:help/images/verb_filters_person.png"></li>
<li>Filters are applied alphabetically by verbal feature.  So Mood first, then Person, then Tense, and finally Voice.  So if you use the same filtering method, say text color, for two different types of verbal feature, one will overwrite the other.  If you made Present Tense verbs green, but also made Active Voice verbs red, then Present Active verbs will be red, because Voice is applied after Tense.  So if you want to filter on multiple verbal features (like the "Default" filter does on Tense and Mood) you need to use separate filtering methods for each.  Color, Highlight, and Formatting can all live on the same verb, but text cannot have multiple colors simultaneously.</li>
<li>There are three levels of highlighting in the program, and they are always applied in the same order.  So if you are using text highlighting for filtering but not seeing it, check that it is not being over written by another level of highlighting.  Verb filter highlights are always applied to the text first.  Then any user defined highlights are applied.  Then finally any selected words are highlighted.  Notice in the following image how my yellow highlight over verse 3 in Jude overwrites all the Active verb highlights.  They are still there, but you will have to remove the custom highlighting on the verse to see them.</li>
<li type="skip"><img src="binfile:help/images/verb_filters_jude_over-hilited.png"></li></ul>
